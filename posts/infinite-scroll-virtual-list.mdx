### Introduction

  <p>
    Data is at the core of most web and mobile applications. From <strong>news feeds</strong> on
    social media to <strong>product catalogs</strong> on e-commerce platforms and{' '}
    <strong>data analytics dashboards</strong>, much of the UI revolves around displaying lists of
    data.
  </p>
  <p>
    It’s safe to say that{' '}
    <strong>90% of web development work involves displaying and managing these lists</strong>.
    However, as data grows, rendering thousands of items at once can{' '}
    <strong>reduce performance</strong>, leading to <strong>slow experiences</strong>. This is where{' '}
    <strong>Infinite Scroll and Virtual List</strong> come into play.
  </p>
</div>

<div>
### Infinite Scroll
  <p>
    What happens when users want to view a massive dataset? Clearly, you can’t throw them a list containing <strong>millions of records</strong>. But why?
  </p>

<h4 className='font-semibold'>The Problem with Loading All Data at Once</h4>
<ul className='list-disc pl-4'>
  <li>
    <strong>Browser Overload:</strong> Displaying thousands or millions of items
    in the DOM leads to:
    <ul className='list-disc pl-4'>
      <li>
        High <strong>RAM usage</strong>.
      </li>
      <li>
        The browser constantly <strong>repainting</strong> and{' '}
        <strong>reflowing</strong> the DOM.
      </li>
      <li>
        <strong>Laggy and slow</strong> experiences.
      </li>
    </ul>
  </li>
  <li>
    <strong>Network Performance Issues:</strong>
    <ul className='list-disc pl-4'>
      <li>
        Sending an API request to fetch{' '}
        <strong>millions of records at once</strong> will:
        <ul className='list-disc pl-4'>
          <li>
            <strong>Take a long time to load</strong>.
          </li>
          <li>
            <strong>Consume significant bandwidth</strong>.
          </li>
          <li>
            <strong>Risk timeout errors</strong>.
          </li>
        </ul>
      </li>
      <li>
        Most users <strong>don’t scroll through all the data</strong>, so
        loading everything upfront is <strong>unnecessary</strong>.
      </li>
    </ul>
  </li>
  <li>
    <strong>Poor User Experience:</strong>
    <ul className='list-disc pl-4'>
      <li>
        Users have to <strong>wait longer</strong> before seeing content.
      </li>
      <li>
        Especially on <strong>mobile devices</strong> with{' '}
        <strong>limited memory</strong>, this can cause{' '}
        <strong>app crashes</strong>.
      </li>
    </ul>
  </li>
</ul>

  <h4 className="font-semibold">What About Pagination?</h4>
  <ul className="list-disc pl-4">
    <li>
      <strong>Pagination</strong> requires users to click a <code>"Next"</code> button to load more content.
    </li>
    <li>
      This works well for <strong>data tables</strong>, but it <strong>disrupts the flow of Browse</strong>.
    </li>
    <li>
      For apps like <strong>TikTok, Facebook, and Shopee</strong>, <strong>natural scrolling</strong> is more important than repeatedly clicking <code>"Next"</code>.
    </li>
  </ul>
  <p>
    <strong>Infinite Scroll solves this by:</strong>
  </p>
  <ul className="list-disc pl-4">
    <li><strong>Automatically loading new data as users scroll.</strong></li>
    <li><strong>Maintaining a seamless Browse experience.</strong></li>
    <li><strong>Being ideal for social media feeds, product catalogs, and news pages.</strong></li>
  </ul>
</div>

### Virtual Scroll (Virtual List)

<h4 className='font-semibold'>The Problem with Infinite Scroll</h4>
<p>
  Infinite Scroll <strong>loads data in chunks</strong>, but it still has a{' '}
  <strong>major drawback</strong>:
</p>
<ol className='compact-list'>
  <li>
    <strong>DOM Overload Over Time</strong>
    <ul className='list-disc pl-4'>
      <li>As the list expands, the number of DOM elements increases.</li>
      <li>
        The browser <strong>consumes more RAM</strong>, causing{' '}
        <strong>lag</strong>.
      </li>
    </ul>
  </li>
  <li>
    <strong>Expensive Reflow & Repaint</strong>
    <ul className='list-disc pl-4'>
      <li>
        More elements mean the browser has to{' '}
        <strong>recalculate layouts (reflow)</strong> constantly.
      </li>
      <li>
        This <strong>reduces performance</strong> and{' '}
        <strong>causes lag</strong>.
      </li>
    </ul>
  </li>
  <li>
    <strong>Unnecessary Resource Consumption</strong>
    <ul className='list-disc pl-4'>
      <li>
        Most users <strong>don’t scroll through the entire list</strong>, but
        the browser still keeps all elements in the DOM.
      </li>
      <li>
        This leads to <strong>wasted memory and resources</strong>.
      </li>
    </ul>
  </li>
</ol>

{" "}

<h4 className='font-semibold'>How Virtual Scroll Solves the Problem</h4>
<p>
  <strong>Virtual Scroll (or Virtual List)</strong> only renders the elements{' '}
  <strong>visible in the viewport</strong>, while removing the rest from the
  DOM.
</p>

  <h4 className="font-semibold">Why Not Just Hide Elements with CSS?</h4>
  <p>
    You might think that <strong>hiding elements (<code>display: none</code>) or using <code>content-visibility: hidden</code></strong> is enough. However, this <strong>DOES NOT reduce resource usage</strong> because:
  </p>
  <ul className="list-disc pl-4">
    <li><strong>Elements still exist in the DOM</strong>.</li>
    <li><strong>The browser keeps them in memory</strong>, not freeing up resources.</li>
    <li><strong>Reflow/repaint costs remain</strong>, still causing lag.</li>
  </ul>
  <p>
    <strong>Virtual Scroll helps:</strong><br />
    ✔ Only render <strong>visible elements</strong>.<br />
    ✔ Render new elements <strong>as needed</strong> when scrolling.<br />
    ✔ Remove elements that scroll out of view <strong>from the DOM</strong>.
  </p>
</div>

<div>
### Edge Cases When Implementing

<h4 className='font-semibold'>Dynamic Height Items</h4>
<ul className='list-disc pl-4'>
  <li>
    When the list contains items with varying heights, calculating scroll
    positions becomes complex.
  </li>
  <li>
    Use an <strong>Item Size Estimator</strong> or{' '}
    <strong>Position Indexing</strong>.
  </li>
</ul>

<h4 className='font-semibold'>Scrolling to a Specific Item</h4>
<ul className='list-disc pl-4'>
  <li>
    When jumping to a specific item (e.g., <code>"Scroll to product #500"</code>
    ), you need to:
    <ul className='list-disc pl-4'>
      <li>Determine the exact position of the item.</li>
      <li>
        Calculate the <strong>correct offset</strong>.
      </li>
    </ul>
  </li>
</ul>

  <h4 className="font-semibold">Horizontal Scrolling (Horizontal Virtual Scroll)</h4>
  <ul className="list-disc pl-4">
    <li>
      Useful for <strong>horizontally scrollable lists</strong>, such as:
      <ul className="list-disc pl-4">
        <li><strong>Video lists (YouTube, Netflix).</strong></li>
        <li><strong>Large data tables.</strong></li>
      </ul>
    </li>
    <li>Optimize using <strong>CSS Grid or Flexbox</strong>.</li>
  </ul>
</div>
